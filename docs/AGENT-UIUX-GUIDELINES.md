# Agent UIUX 设计规范

本文用于约束后续 agent 在 PiDownloader 项目中的界面实现方式，避免出现逻辑、数据、UI 混写，或后续迭代时遗忘关键设计规范。

## 1. 分层原则

1. 逻辑层只负责状态变更、命令调用、数据转换，不直接拼装视觉结构。
2. 数据层负责持久化、IPC、后端配置读写，不参与组件视觉决策。
3. UI 层只接收受控 props，不直接访问后端或写业务副作用。
4. 公共控件必须优先抽到 `frontend/src/components/ui/`，页面文件只负责组合。
5. 具有结构复用价值的页面片段应抽到 `frontend/src/components/settings/` 或同级领域目录。

## 2. 组件复用要求

1. 滚动区域统一使用公共 `ScrollArea`，不要在页面里直接重复写浏览器滚动条样式。
2. 开关统一使用公共 `Switch`，必须受控，使用 `checked` 和 `onCheckedChange`。
3. 滑条统一使用公共 `Slider`，必须受控，显示当前值时优先使用文案或数值槽位，不把业务文案写进组件内部。
4. 输入框、字段块、设置卡片应尽量通过基础组件或领域 primitives 组合，不在页面里重复同类样式。
5. 页面级组件禁止私自定义一套新的尺寸体系，必须优先走 token 或 CSS 变量。

## 3. Token 与尺寸规范

1. 宽度、间距、标题栏高度、图标尺寸优先来自 `frontend/src/core/ui-tokens.ts`。
2. 圆角、边框、前景色、背景色、强调色优先来自全局 CSS 变量，如 `--radius`、`--border`、`--card`、`--primary`。
3. 单个页面不得硬编码一整套视觉参数；零星必要值必须有上下文理由。
4. 同一层级组件应共享字号和行高，避免局部自定义造成节奏紊乱。

## 4. 文本与排版规范

1. 交互控件正文默认使用 `text-sm` 或 `text-base`，避免过小字号。
2. 标题必须有明确层级：页面标题、区块标题、字段标题、说明文字。
3. 文字需保证像素对齐和行高稳定，避免文本与图标中心错位。
4. 长段说明使用 `leading-6` 左右的舒适行高，不要堆密集说明块。
5. 中文文案保持自然简洁，避免混用乱码、占位符式英文或机器直译。

## 5. 安全边距与布局规范

1. 页面边缘必须保留安全边距，避免内容贴边。
2. 左右分栏布局中，导航栏与内容区必须有明确视觉分隔。
3. 组件之间使用稳定节奏的垂直间距，不要局部忽大忽小。
4. 卡片内部需要保证标题、正文、操作区三段式呼吸感。
5. 设置类页面优先使用左导航右内容结构，右侧内容需要可滚动，导航区保持稳定宽度。

## 6. 可用性规范

1. 所有交互控件必须提供 hover、focus、disabled 状态。
2. 开关、滑条、输入框必须可键盘操作。
3. 页面反馈要明确，包括加载中、保存中、保存成功、错误提示。
4. 默认值、单位、限速含义等必须写清楚，避免用户猜测。
5. 表达“空值”的控件要明确说明，例如“留空表示不限速”。

## 7. 桌面窗口规范

1. 自定义标题栏必须显式标记拖拽区域。
2. 关闭、最小化、最大化、独立设置窗口等操作要通过明确命令桥接，不把窗口 API 调用散落到任意组件。
3. 需要禁止复制的纯展示型 UI 页面，可以在页面根节点禁用选择；输入框和可编辑区域除外。
4. 多窗口页面的路由必须清晰，例如主窗口、悬浮窗、设置窗分别独立。

## 8. 代码组织规范

1. `components/ui/` 放通用控件。
2. `components/settings/` 放设置页特有的结构化 primitives 与页面组合件。
3. `core/store/` 放状态管理。
4. `core/bridge/` 放 Tauri 或后端命令桥。
5. `core/locale.ts` 放界面文案，不在页面中散落硬编码中文。

## 9. 禁止项

1. 禁止把后端调用、状态拼装、视觉 JSX 混在一个难以复用的大组件里。
2. 禁止在页面中直接复制滚动条、滑条、开关样式实现。
3. 禁止在没有 token 或变量依据时大量硬编码 `px` 数值。
4. 禁止新增乱码文案、异常符号占位、未校验的设计注释。
5. 禁止为了省事让设置页直接操作 DOM 来保存业务状态。

## 10. 交付检查清单

1. 是否已将重复控件抽成公共组件？
2. 是否保证逻辑、数据、UI 三层职责清晰？
3. 是否遵守全局 token、间距、字号、图标尺寸规范？
4. 是否提供了加载、保存、错误反馈？
5. 是否考虑桌面窗口拖拽、关闭行为、滚动、不可复制等桌面端细节？
