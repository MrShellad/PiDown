PiDownloader 下载桥接

开启后，扩展会把符合条件的 HTTP/HTTPS 下载交给 PiDownloader。

Native Host 扩展固定使用 com.pidownloader.bridge。请在当前系统中注册同名 Native Messaging Host。

基础行为

控制接管失败时的回退方式，以及桥接期间的下载处理策略。

过滤规则

按扩展名和文件大小控制哪些下载会被桥接到 PiDownloader。

常用预设

留空表示不限制。多个扩展名用英文逗号分隔，输入时可省略点号。

常用预设

命中黑名单时不会桥接。适合排除临时文件、浏览器中间文件或特殊后缀。

大于 0 时，仅桥接已知大小且不小于该值的下载。留空或填 0 表示不限制。

连接说明

扩展本身跨平台可用；Native Host 需要按当前系统完成注册。

Windows 可使用仓库中的 native-host/register-windows.example.ps1 作为开发示例。
macOS / Linux 请将同名 Native Host manifest 放到浏览器支持的 Native Messaging 路径，并指向本机可执行文件。
